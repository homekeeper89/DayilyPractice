옵저버 패턴 : 행위패턴의 한 종류

- 생성패턴(싱글턴과 같은)과는 다른 개념.
- 행위패턴이 주목하는 것은 객체 간의 상호작용
- 객체는 상호작용 하지만 느슨하게 결합되어있다.

객체(subject)는 자식(observer)의 목록을 유지 한다.
객체가 자식의 메서드를 호출할때마다 자식은 이를 알린다.

예) 웹서비스가 사용자의 계정상태를 확인, 이메일을 발송하는 서비스가 존재한다고 할 경우
사용자가 가입을 하면 이웹서비스는 이메일 서비스의 메소드를 호출, 인증 이메일을 발송한다.
계정 인증이 됐지만 포인트가 부족한 경우 사용자에게 이를 알리는 이메일을 발송

여기까지 적었을때 궁금한점) 객체(subject)는 자식을 항상 지켜보고 있는건가? 그렇다면 true/false를 통해서 알아채리는건가?

예) 블로그 구독 후 새로운 글이 등록되면 구독자는 알림을 받는다.

- 객체간 일대다(1:N) 관계 형성
- 객체(subject) 핵심 부분을 캡슐화
