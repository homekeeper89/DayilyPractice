프록시
- 요청자와 공급자 사이의 중재자
- 프록시 클래스는 객체의 인터페이스 역할을 한다
예)
- 영화사 - 에이전트(프록시) - 배우
- 에이전트에게 배우가 바쁜지 물어본다

특정 객체의 대리 객체를 제공해 접근을 제어
분산 구조 지원 위한 레이어 또는 인터페이스 제공
의도하지 않은 케이스로부터 객체 보호

예) 물건을 백화점에서 살때 현금카드를 통해서 살 경우
클라이언트부터 작성하게 되면
make_payment() 를 사용해 결제
__init__()는 프록시를 생성하는 메서드
make_payment는 내부적으로 proxy 메서드를 호출해 금액을 지불
결제 성공시 __del__() 메서드 호출

- 프록시는 RealSubject를 보호하는 역할
- 제한된 접근만을 가능하게 함
- 성능 이슈가 나올 수 있음
- RealSubject 코드를 수정하지 않고 기능을 추가 할 수 있다.
